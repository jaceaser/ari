# 04 — QA

You are running the QA phase of the ARI Codex pipeline.

## Task

Run `qa_check.py`, review the results, and fix any failures by re-running the appropriate chapter or case study workers.

---

## Step 1 — Run QA

```bash
cd apps/codex/course-guides/{{config.codex_slug}}
python3 ../../pipeline/scripts/qa_check.py --config config.yaml
```

---

## Step 2 — Show the full qa_report.md

Display the complete contents of `qa_report.md`.

If all checks pass: report ✅ QA PASSED and stop.

---

## Step 3 — Address failures

For each FAIL or MISSING item in the report:

### If MISSING (output file doesn't exist):
The chapter or case study worker was never run or failed silently.  
Re-invoke prompt `02_chapter_worker.md` for this item.

### If FAIL (output < 60% of source word count):
The worker summarized instead of preserving content.  

Re-invoke `02_chapter_worker.md` with this additional instruction prepended:

```
IMPORTANT: The previous output was rejected by QA for being too condensed.
Source word count: {{source_words}}
Previous output word count: {{output_words}}
Required minimum: {{min_words}} words (60% of source)

You MUST produce at least {{min_words}} words of output.
Do not summarize any section. Every case study, every numbered step, every sub-section 
must appear in full. If you are uncertain about a section, include the full text from 
the source rather than a shortened version.
```

---

## Step 4 — Re-run QA

After fixing all failures, re-run `qa_check.py` and confirm all checks pass.

If failures persist after 3 attempts on a single item, escalate to the operator with:
- The specific chapter/case study number and title
- The source word count
- The best output achieved so far
- What content appears to be getting condensed

---

## Step 5 — Final report

Show:
- Total checks: X
- Passing: X
- Any items that required re-running (and how many attempts)
- Confirmation that `course.config.yaml` has `frontPageQuote` set
- Confirmation that `overview.md` is present and non-empty
- Total topics/ files: X (should be 42 for Fractured Equity)
- Total case-studies/ files: X (should be 100 for Fractured Equity)

The codex is ready to deploy once all checks pass.
