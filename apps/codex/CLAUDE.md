# ARI Codex — Claude Orientation

## What this app is

A Next.js 16 + React 19 interactive course platform. Content is stored as markdown files
with YAML frontmatter. The UI is a full-screen slide-based presentation player (swipe +
keyboard navigation). Multiple courses ("codices") can live in the same app — each in its
own `course-guides/{slug}/` folder.

## Content system

All content is markdown with YAML frontmatter.  Read `types/codex.ts` for all entity shapes.

Key entity types:
- **topic** — a chapter (42 for Fractured Equity, one per chapter)
- **case-study** — a cliff-note scenario (100 for Fractured Equity)
- **pathway** — step-by-step decision guide
- **operator-card** — field checklist
- **glossary** — term definition

The content loader (`lib/content-loader.ts`) reads all markdown from `course-guides/{slug}/`
at request time using `gray-matter` + `remark`. No database. No build step for content.

---

## Slide system

`lib/slide-builder.ts` assembles a flat `Slide[]` array from a `SerializedCourse`:
1. Hero slide (landing page)
2. Chapter-intro slide ("Part I: The Strategy")
3. One topic slide per topic (in order field)
4. Chapter-intro slide ("Part II: Case Studies")
5. One case-study slide per case study
6. Chapter-intro slide ("Part III: Pathways")
7. One pathway slide per pathway
8. Glossary slide
9. Knowledge Map slide

---

## Landing page / Hero slide

`components/codex/slides/HeroSlide.tsx`

The hero displays:
- Course title (from `course.config.yaml: title`)
- Course description (from `course.config.yaml: description`)
- Front-page pull quote (from `course.config.yaml: frontPageQuote` + `frontPageQuoteAttribution`)
- Curriculum part cards (topic count, case study count, pathway count)
- "Press to Begin" CTA (large — minimum 1.25rem font, prominent gold button)
- "Browse Curriculum" secondary CTA

The `frontPageQuote` field was added to `CourseConfig` in `types/codex.ts`.  
If a course doesn't have a front-page quote, the blockquote is simply not rendered.

## Content pipeline

To rebuild a course from its manuscript:

```bash
# In the course-guides/{slug}/ directory:
python3 ../../pipeline/scripts/extract_manuscript.py --config config.yaml
# Then invoke the orchestrator prompt to run chapter workers:
# pipeline/prompts/00_orchestrator.md
# Then validate:
python3 ../../pipeline/scripts/qa_check.py --config config.yaml
```

See `pipeline/README.md` for the full workflow.

### Adding a new codex

1. `mkdir -p course-guides/{new-slug}/{topics,case-studies,operator-cards,glossary,pathways}`
2. Drop the manuscript `.docx` in that directory
3. Copy `pipeline/config.example.yaml` → `course-guides/{new-slug}/config.yaml` and fill in
4. Copy `course-guides/fractured-equity/course.config.yaml` as a template and update
5. Invoke `pipeline/prompts/00_orchestrator.md` with the config path
6. Deploy: `az acr build -r ariprodacr -t ari-codex:latest .`

## QA rule

Any topic output file must contain ≥ 60% of the word count of its source chapter.
Any case-study output file must contain ≥ 60% of the word count of its source cliff note.
This rule prevents the most common failure mode: AI workers summarizing instead of transcribing.

## Deployment

Docker container on Azure App Service, same pattern as `apps/api` and `apps/web`.

```bash
# From apps/codex:
az acr build -r ariprodacr -t ari-codex:latest .
az webapp restart -g rg-ari-prod -n ari-codex
```

Content is read from the filesystem at runtime — adding a new course requires a redeploy.
